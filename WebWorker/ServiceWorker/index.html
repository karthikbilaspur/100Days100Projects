How to Test Offline Functionality

  1. Ensure you visit this page at least once while **online** to allow the Service Worker to cache files.
  2. Open your browser's Developer Tools (F12 or Cmd+Option+I).
  3. Go to the **"Network"** tab and select the **"Offline"** checkbox or option from the dropdown menu (usually near the top). Alternatively, simply turn off your device's Wi-Fi or data connection.
  4. **Refresh this page.**
  5. Observe: The page should still load from the cache, and the status indicator above should update to "Offline"!
  6. Check the **"Application"** tab in Developer Tools for Service Worker logs and cached files.

What is a Web Worker?

A Web Worker is a JavaScript script executed by an HTML page that runs in the background, independently of other scripts, without affecting the performance of the page. They are particularly useful for performing CPU-intensive tasks without blocking the user interface.

Why Service Workers?

Service Workers are a type of Web Worker that primarily act as a programmable network proxy. They sit between your web application and the network, intercepting requests and allowing you to programmatically control how network requests are handled. This enables features like rich offline experiences, push notifications, and background data synchronization.
